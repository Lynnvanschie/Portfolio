Project Web

Situatie

Ik heb deze opdracht in het twee jaar van CMD uitgevoerd om het vak Project Web te halen. Hierbij moesten we een interactieve reisgids ontwerpen voor een specifiek gedeelte van een reis.

Taak

De opdracht was het maken van een prototype in bijvoorbeld adobe xd. Het prototype moest multidevice zijn en specifiek zijn van een gedeelte van de reis, bijvoorbeeld het boeken of activiteiten vinden.

Activiteiten

Ik ben begonnen met wat onderzoek doormiddel van een customer journey map en APIs. Daarna heb ik meteen een eerste prototype gemaakt. De focus ligt bij dit project niet op onderzoek, maar op een iteratief proces. Waarbij je een prototype maakt, daar feedback op krijgt en daar weer een vernieuwd prototype uit maakt. Hier krijg je weer feeback op enzovoorts.

Resultaat

Uiteindelijk heb ik een prototype gemaakt waarbij mensen op een stedentrip met vrienden activiteiten van te voren op kan zoeken en op kan slaan. Eenmaal op reis kunnen ze de activiteiten op een kaart zien en de routes naar de activiteiten genereren. Achteraf had ik nog wel meer aandacht aan de vormgeving willen geven.

Reflectie

Ik vind dit een goed voorbeeld voor evalueren, omdat ik elke week feedback heb gekregen op mijn prototype, wordt je elke week gedwongen om kritisch naar jouw eigen werk te kijken. Ook heb ik zelf actief feedback gezocht omdat ik merkte dat hoe meer feedback je ontvang hoe beter jouw uiteindelijke product is. Dit is iets dat ik meeneem naar volgende projecten.
